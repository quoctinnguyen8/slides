OOP Trong Python: Điểm Chung Và Khác

  • Điểm chung: Vẫn có class, object, kế thừa, đa hình, trừu tượng.
  • Khác biệt lớn: Python không có access modifier cứng, dùng quy ước tên chung.
  • self tường minh: Method bắt buộc có tham số đầu tiên là self.
  • Duck typing: Ưu tiên hành vi của object, không quá phụ thuộc kiểu khai báo.
  • Mọi thứ là object: function/class/module đều có thể truyền như dữ liệu.

Duck Typing Trong Python

Nguyên tắc: “Nếu nó đi như vịt và kêu như vịt, thì xem nó như vịt”.

  • Không bắt buộc object phải cùng class cha mới dùng được chung hàm.
  • Chỉ cần object có method/hành vi mà hàm cần gọi.
  • Giúp code linh hoạt, giảm phụ thuộc theo kiểu dữ liệu cụ thể.
  • Khi cần an toàn hơn: kết hợp type hints và test.
class Dog:
    def speak(self): 
        return "Gâu"

class Robot:
    def speak(self):
        return "Xin chào"

def talk(obj):
    print(obj.speak())

talk(Dog())
talk(Robot())

Class Và Object

Class là bản thiết kế. Object là thực thể được tạo ra từ class đó.

  • class Student: định nghĩa khuôn mẫu học viên.
  • an = Student(...) tạo object cụ thể.
  • Method luôn khai báo self làm tham số đầu tiên.
  • Không dùng từ khóa new để khởi tạo object.
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        return f"Tôi là {self.name}, {self.age} tuổi"

an = Student("An", 20)
print(an.introduce())

Class Property Và Object Property

  • Class property: thuộc về class, dùng chung cho mọi object.
  • Object property: thuộc từng object, mỗi object có 1 bản riêng.
  • Sửa class property sẽ ảnh hưởng tất cả object (nếu chưa bị override ở object).
class Student:
    school = "PTIT"          # class property

    def __init__(self, name):
        self.name = name     # object property

a = Student("An")
b = Student("Bình")

Student.school = "UTE"
print(a.school, b.school)  # UTE UTE

Quy Ước Property Và Dunder Methods

  • Public: name — truy cập công khai.
  • Protected (quy ước): _score — dùng nội bộ/class con.
  • Private (name mangling): __password sẽ được đổi tên thành _ClassName__password.
  • Dunder phổ biến:
  • __init__: khởi tạo object.
  • __str__: hiển thị object khi print.
  • __repr__: hiển thị kỹ thuật cho debug.
  • __len__: cho phép dùng hàm len().
  • __eq__: tùy biến so sánh bằng.
class Student:
    def __init__(self, name, score):
        self.name = name          # public
        self._score = score       # protected
        self.__token = "xyz"      # private

    def __str__(self):
        return f"Student({self.name})"

    def __eq__(self, other):
        return self.name == other.name

Phân Biệt @staticmethod Và @classmethod

  • @staticmethod: không nhận self hay cls.
  • Phù hợp cho hàm tiện ích liên quan ngữ nghĩa class.
  • @classmethod: nhận cls, thường dùng làm factory method.
  • Classmethod tạo object theo nhiều cách khởi tạo khác nhau.
  • Rule nhanh: cần truy cập class -> classmethod; không cần -> staticmethod.
class User:
    def __init__(self, name, age):
        self.name, self.age = name, age

    @classmethod
    def from_birth_year(cls, name, year):
        return cls(name, 2026 - year)

    @staticmethod
    def is_adult(age):
        return age >= 18

@dataclass: Khi Nào Dùng Và Lợi Ích

@dataclass phù hợp với class chủ yếu để lưu dữ liệu (DTO, config, response model).

  • Tự sinh __init__, __repr__, __eq__.
  • Giảm boilerplate so với viết class thuần thủ công.
  • Kết hợp type hints giúp IDE kiểm tra tốt hơn.
  • Dùng frozen=True nếu muốn khóa object.
  • Lưu ý class property: dùng ClassVar để đánh dấu thuộc tính cấp class.
  • Lưu ý object property mutable: dùng field(default_factory=list) thay vì [] để tránh dùng chung dữ liệu.
from dataclasses import dataclass, field
from typing import ClassVar

@dataclass
class Product:
    tax_rate: ClassVar[float] = 0.1  # class property
    name: str
    price: float
    tags: list[str] = field(default_factory=list)

p = Product("Bàn phím", 590000)
p.tags.append("gaming")

Trụ Cột 1: Đóng Gói (Encapsulation)

  • Ẩn dữ liệu nội bộ để tránh thao tác sai trực tiếp.
  • Kết hợp @property và setter để kiểm tra dữ liệu.
  • Thiết kế method class rõ ràng: cái nào cho phép gọi từ bên ngoài.
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self._balance = balance

    @property
    def balance(self):
        return self._balance

    def deposit(self, amount):
        if amount > 0: self._balance += amount

Trụ Cột 2: Kế Thừa (Inheritance)

  • Class con tái sử dụng và mở rộng logic từ class cha.
  • Dùng super() nhất quán trong constructor/method ghi đè.
  • Chỉ dùng kế thừa khi quan hệ thật sự là is-a.
class Animal:
    def speak(self): 
        return "Âm thanh chung"

class Dog(Animal):
    def speak(self): 
        return "Gâu gâu"

class Cat(Animal):
    def speak(self):
        return "Meo meo"

Trụ Cột 3-4: Đa Hình Và Trừu Tượng

  • Đa hình: Cùng method name, hành vi khác nhau theo object.
  • Trừu tượng: Dùng ABC để ép class con triển khai method bắt buộc.
  • Giúp code mở rộng tốt và giảm if/else theo type.
from abc import ABC, abstractmethod

class Gateway(ABC):
    @abstractmethod
    def pay(self, amount): 
        pass

class Momo(Gateway):
    def pay(self, amount): 
        return f"Momo {amount}"

class Bank(Gateway):
    def pay(self, amount):
        return f"Bank {amount}"

Đa Kế Thừa, super() Và MRO

  • Python cho phép đa kế thừa (multiple inheritance).
  • Thứ tự gọi method theo MRO (Method Resolution Order).
  • Kiểm tra bằng ClassName.mro().
class A: 
    def log(self): 
        return "A"
class B(A):
    def log(self): 
        return "B->" + super().log()

class C(A): 
    def log(self): 
        return "C->" + super().log()

class D(B, C):
    pass

print(D().log())  # B->C->A

Bài Tập

Đề bài: Xây hệ thống quản lý nhân sự với class cha Employee gồm name, base_salary và method calculate_salary().

Tạo FullTimeEmployee và PartTimeEmployee, ghi đè method tính lương và in kết quả theo đa hình.

Part-time: lương tính theo giờ; Full-time: lương cơ bản cộng thêm phụ cấp.

Gợi Ý Triển Khai Bài Tập

  • Tạo class cha Employee có constructor chung và calculate_salary().
  • Ở class con dùng super().__init__() để tái sử dụng logic khởi tạo.
  • Ghi đè method tính lương theo từng loại nhân sự.
  • Duyệt list nhân sự và gọi cùng một method để thể hiện đa hình.

Đáp Án Mẫu: Bài Tập Kế Thừa

class Employee:
    def __init__(self, name, base):
        self.name, self.base = name, base
    def calculate_salary(self): return self.base

class FullTimeEmployee(Employee):
    def __init__(self, name, base, bonus):
        super().__init__(name, base)
        self.bonus = bonus
    def calculate_salary(self):
        return self.base + self.bonus

class PartTimeEmployee(Employee):
    def __init__(self, name, rate, hours):
        super().__init__(name, 0)
        self.rate, self.hours = rate, hours
    def calculate_salary(self):
        return self.rate * self.hours

staffs = [FullTimeEmployee("An", 12_000_000, 2_000_000), PartTimeEmployee("Bình", 120_000, 80)]
for emp in staffs:
    print(emp.name, emp.calculate_salary())
