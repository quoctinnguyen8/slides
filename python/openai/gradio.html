Từ giao diện cơ bản đến ứng dụng AI có nhiều tab, state và triển khai lên Hugging Face Spaces.

  • Gradio Docs (https://gradio.app/docs): Tài liệu chính thức hướng dẫn component, Blocks, ChatInterface, state và triển khai.

  • Python Docs (https://docs.python.org/3/): Môi trường ảo, package management và xử lý dữ liệu cơ bản.

Gradio Là Gì? Vì Sao Nên Dùng?

Gradio là gì?

Gradio là thư viện Python giúp tạo giao diện web cho mô hình AI hoặc hàm Python chỉ với vài dòng code.

Vì sao cần dùng?

  • Prototype cực nhanh để demo ý tưởng với đội ngũ hoặc khách hàng.
  • Không cần viết frontend phức tạp (HTML/CSS/JS) ở giai đoạn đầu.
  • Dễ tích hợp mô hình ML/LLM, API nội bộ và dữ liệu thực tế.

Ưu điểm

  • Tạo app tương tác trong thời gian ngắn.
  • Nhiều component dựng sẵn: text, ảnh, audio, chat, file.
  • Hỗ trợ chia tab, state, event rõ ràng.
  • Triển khai nhanh lên Hugging Face Spaces.

Hạn chế

  • Khả năng tùy biến giao diện sâu không linh hoạt bằng framework frontend chuyên biệt.
  • Chỉ phù hợp với các ứng dụng đơn giản, không nên dùng cho hệ thống phức tạp.

Giao Diện Cơ Bản Với Gradio

Cài đặt package: pip install gradio

Giải thích tham số:

  • fn: Hàm xử lý chính. Nhận dữ liệu từ input và trả về dữ liệu cho output.
  • inputs: Component đầu vào (Textbox, Image, Audio...). Có thể là 1 component hoặc danh sách.
  • outputs: Component hiển thị kết quả. Kiểu trả về của hàm cần khớp với output.
  • title: Tiêu đề lớn của ứng dụng trên giao diện.
  • description: Mô tả ngắn hướng dẫn người dùng cách sử dụng app.
  • examples: Danh sách dữ liệu mẫu để bấm thử nhanh.
import gradio as gr

def greet(name):
    return f"Xin chào {name}!"

app = gr.Interface(
    fn=greet,
    inputs=gr.Textbox(label="Nhập tên"),
    outputs=gr.Textbox(label="Kết quả"),
    title="Ứng dụng Gradio đầu tiên",
    description="Nhập tên để nhận lời chào",
    examples=["An", "Bình"]
)

app.launch()

Giao Diện Chat Với ChatInterface

ChatInterface là cách nhanh nhất để tạo chatbot trong Gradio.

  • fn(message, history): Hàm xử lý khi nhận được dữ liệu từ ô nhập liệu; history là lịch sử hội thoại.
  • title và description: Tiêu đề và mô tả.
  • submit_btn: Đổi tên nút gửi tin nhắn.
import gradio as gr

def chat_fn(message, history):
    return f"Bạn vừa nói: {message}"

demo = gr.ChatInterface(
    fn=chat_fn,
    title="Chatbot Gradio",
    description="Ví dụ ChatInterface cơ bản",
    submit_btn="Gửi"
)

demo.launch()

Các Control Cơ Bản Trong Gradio

Control Mục đích Ví dụ sử dụng
Textbox / Number Nhập văn bản hoặc số từ người dùng. Prompt, tên, tuổi, số lượng.
Checkbox / Radio Bật tắt tuỳ chọn hoặc chọn một giá trị. Bật streaming, chọn chế độ trả lời.
Dropdown / Slider Chọn giá trị trong danh sách hoặc dải số. Model, temperature, max_output_tokens.
File / Image / Audio Nhận dữ liệu đa phương tiện để AI xử lý. Upload hóa đơn, ảnh sản phẩm, audio ghi âm.

Bài Tập: Chatbox Với OpenAI API

Yêu cầu: Dùng gr.ChatInterface tạo chatbot trả lời bằng OpenAI API. Đọc API key từ file .env, thêm system prompt ngắn và hiển thị lịch sử hội thoại.

Gợi ý: Cài openai, python-dotenv. Trong hàm chat, gọi API bằng model bất kỳ và trả về chuỗi nội dung.

Gợi Ý Giải Bài Tập: Chatbox OpenAI

  • Tách logic và giao diện: Viết hàm xử lý gọi OpenAI trong một module riêng (ví dụ chat_service.py), file main.py chỉ phụ trách UI Gradio.
  • Bảo mật key: Dùng .env + load_dotenv(), không hard-code key trong code.
  • Xử lý history: Duyệt lịch sử chat từ Gradio và chuyển về định dạng messages (user/assistant) trước khi gọi API.
  • Thiết kế hàm trả về: Hàm xử lý trả về chuỗi văn bản để ChatInterface hiển thị ngay phản hồi.
  • Package cần thiết: pip install gradio openai python-dotenv.

Code Giải Bài Tập (Tách Module)

# chat_service.py 
from openai import OpenAI
from dotenv import load_dotenv

load_dotenv()
client = OpenAI()

def chat_with_openai(message, history):
    system_prompt = "Bạn là trợ lý AI trả lời ngắn gọn, dễ hiểu."

    # Thêm lịch sử chat vào messages
    messages = []
    for msg in history:
        messages.append({"role": msg["role"], "content": msg["content"]})

    messages.append({"role": "user", "content": message})

    response = client.responses.create(
        model="gpt-5-mini",
        instructions=system_prompt,
        input=messages,
        max_output_tokens=1000
    )
    return response.output_text
# main.py
import gradio as gr
from chat_service import chat_with_openai

demo = gr.ChatInterface(
    fn=chat_with_openai,
    title="Chatbox OpenAI + Gradio",
    description="Bài tập thực hành dùng OpenAI API",
    examples=["Gradio là gì?", "Cho tôi 3 mẹo học Python"]
)

demo.launch()

Chạy app: python main.py hoặc gradio main.py

Xây Dựng Layout Cơ Bản Với Blocks()

Vì sao dùng Blocks?

  • Kiểm soát bố cục tốt hơn Interface/ChatInterface.
  • Định nghĩa rõ component nào tương tác với component nào.
  • Phù hợp ứng dụng nhiều khu vực: nhập liệu, kết quả, cấu hình.

Cách hiểu nhanh:
with gr.Blocks() as app: là khung tổng. Bên trong tạo component, sau đó gắn event bằng .click(), .change()...

import gradio as gr

def to_upper(text):
    return text.upper()

with gr.Blocks() as app:
    gr.Markdown("## Demo Blocks")
    inp = gr.Textbox(label="Nhập nội dung")
    out = gr.Textbox(label="Kết quả")
    btn = gr.Button("Chuyển")
    btn.click(to_upper, inputs=inp, outputs=out)

app.launch()

Chia Tab Trong Gradio

  • Dùng gr.Tab("Tên tab") để chia app thành nhiều khu vực chức năng.
  • Giúp giao diện gọn hơn khi có nhiều form hoặc phần cấu hình.
  • Phù hợp kiến trúc: Tab Chat, Tab Cấu hình, Tab Nhật ký.

Lưu ý: Tab chỉ là bố cục giao diện. Dữ liệu chia sẻ giữa tab vẫn cần quản lý bằng State hoặc biến xử lý phù hợp.

with gr.Blocks() as app:
    with gr.Tab("Chat"):
        chatbox = gr.Chatbot()
        user_inp = gr.Textbox()

    with gr.Tab("Cấu hình"):
        model = gr.Dropdown(
            label="Chọn model",
            choices=["gpt-5.2", "gpt-5-mini"],
            value="gpt-5.2"
        )
        temperature = gr.Slider(
            label="Temperature (độ sáng tạo)",
            minimum=0,
            maximum=2,
            value=1
        )

State Trong Gradio

State dùng để làm gì?

  • Lưu dữ liệu tạm trong phiên làm việc của người dùng.
  • Giữ lịch sử chat, bộ lọc, hoặc kết quả trung gian giữa nhiều event.
  • Giúp app có cảm giác liền mạch thay vì xử lý rời rạc.
  • Lưu ý: Dữ liệu lưu ở State sẽ mất khi đóng hoặc refresh trang.

Tư duy sử dụng:
Input từ component + state cũ → hàm xử lý → output UI mới + state mới.

import gradio as gr

def add_message(message, history):
    history = history or []
    history.append({"role": "user", "content": message})
    history.append({"role": "assistant", "content": "Đã nhận."})
    return history, history

with gr.Blocks() as app:
    state = gr.State([])
    chat = gr.Chatbot(type="messages")
    text = gr.Textbox()
    text.submit(add_message, [text, state], [chat, state])

Bài Tập: Build Chatbox Thủ Công

Mục tiêu: Dùng Blocks + Chatbox + Textbox + Button để tự dựng chatbot, không dùng ChatInterface.

Yêu cầu chi tiết:
1. Tab 1: Chat với vùng hiển thị hội thoại, ô nhập và nút gửi.
2. Tab 2: Cấu hình gồm model, temperature, max_output_tokens.
3. Dùng State để lưu history.
4. Khi bấm gửi ở Tab 1, áp dụng cấu hình hiện tại từ Tab 2 để gọi model.

Tùy Biến Giao Diện Với Row Và Column

  • gr.Row(): xếp component theo chiều ngang.
  • gr.Column(): xếp component theo chiều dọc trong từng vùng.
  • Dễ tạo layout kiểu dashboard: trái là chat, phải là panel cấu hình.
  • Kết hợp tham số scale để chia tỷ lệ chiều rộng linh hoạt.

Gợi ý: Chat panel scale lớn hơn (ví dụ 3), cột cấu hình scale nhỏ hơn (ví dụ 1).

with gr.Blocks() as app:
    with gr.Row():
        with gr.Column(scale=3):
            chat = gr.Chatbot(label="Hội thoại")
            prompt = gr.Textbox(label="Nhập câu hỏi")

        with gr.Column(scale=1):
            model = gr.Dropdown(
                choices=["gpt-5.2"]
            )
            temp = gr.Slider(0, 1, value=0.7)

Gradio Themes: Dùng Theme Có Sẵn

Gradio hỗ trợ nhiều theme có sẵn giúp thay đổi giao diện nhanh mà không cần viết CSS nhiều.

  • gr.themes.Default(): giao diện mặc định, trung tính.
  • gr.themes.Soft(): cảm giác mềm mại, phù hợp app demo.
  • gr.themes.Monochrome(): tối giản, tập trung nội dung.
  • gr.themes.Glass(): phong cách hiện đại, nổi bật.

Lưu ý: Chọn theme phục vụ nội dung, tránh lạm dụng màu sắc gây rối trải nghiệm.

import gradio as gr

with gr.Blocks() as app:
    ...

app.launch(theme=gr.themes.Soft())

Các Khả Năng Nâng Cao Khác Của Gradio

  • Xác thực người dùng: Dùng launch(auth=...) để bảo vệ ứng dụng nội bộ.
  • Nhiều trang / nhiều khu vực: Dùng Tab, TabbedInterface hoặc tách app theo route khi tích hợp backend.
  • Thông báo nổi: Dùng gr.Info(), gr.Warning(), gr.Error() để phản hồi tức thì.
  • Hàng đợi và tác vụ nặng: Dùng queue() để xử lý ổn định khi nhiều người dùng đồng thời.
  • Kiểm soát truy cập API: Cấu hình api_name cho event, bật/tắt endpoint theo nhu cầu triển khai.

Tìm hiểu thêm tại trang tài liệu chính thức của Gradio.

import gradio as gr

def handle(text):
    if not text:
        gr.Warning("Bạn chưa nhập nội dung.")
        return ""
    gr.Info("Đang xử lý yêu cầu...")
    return text.upper()

with gr.Blocks() as app:
    inp = gr.Textbox(label="Nhập nội dung")
    out = gr.Textbox(label="Kết quả")
    btn = gr.Button("Xử lý")
    btn.click(handle,
        inputs=inp, outputs=out,
        api_name="handle_text")

app.queue().launch(auth=[("admin", "123456")])
