Vector Embedding Là Gì?

Vector Embedding là kỹ thuật chuyển đổi các dạng dữ liệu phi cấu trúc (đoạn văn, hình ảnh, âm thanh) thành các mảng số thực (vector).

Có thể hiểu đây là cách "số hóa" ngữ nghĩa của dữ liệu để máy tính có thể hiểu, phân tích và tính toán được.

Ví dụ văn bản gốc:

Tôi thích học AI

Sau khi thực hiện embedding:

[0.21, -0.43, 0.77, ...]

Mỗi con số trong vector đại diện cho một đặc trưng ngữ nghĩa của văn bản.

Ví dụ cụ thể với bộ dữ liệu sau

Embedding: Giải Thích Trực Quan

Biểu diễn dữ liệu với 1 tiêu chí

Biểu diễn dữ liệu với 2 tiêu chí

Biểu diễn dữ liệu với đủ 3 tiêu chí

Mỗi tiêu chí được gọi là một chiều (dimension) trong vector.

Embedding Được Tạo Bởi Model

Vector embedding được tạo ra bằng cách sử dụng các mô hình đã được huấn luyện. Các mô hình này có khả năng đánh giá văn bản dựa trên hàng ngàn tiêu chí (dimensions) khác nhau.

  • Model khác nhau sẽ cho vector khác nhau.
  • Số chiều (dimensions) càng lớn khả năng tìm kiếm càng chính xác, tuy nhiên tốc độ tìm kiếm và chi phí lưu trữ sẽ tăng lên. Chức năng RAG thông thường cần vector có 768-1536 chiều.
  • Lưu ý quan trọng: mỗi dimensions trong vector không có nghĩa rõ ràng với con người. Ngữ nghĩa của câu/đoạn văn nằm ở toàn bộ vector.

Ví dụ tạo vector với OpenAI

import openai

response = openai.embeddings.create(
    input="Your text string goes here",
    model="text-embedding-3-small"
)

print(response.data[0].embedding)
print("Length of embedding:", len(response.data[0].embedding))
# Length of embedding: 1536

Similarity Search Hoạt Động Thế Nào?

  • Similarity search là quá trình tìm kiếm các vector trong database có độ tương đồng cao nhất với vector của câu hỏi.
  • Độ tương đồng được tính bằng cách sử dụng khoảng cách giữa các vector, khoảng cách càng nhỏ thì độ tương đồng càng cao.
  • Với vector 2 chiều, độ tương đồng có thể được biểu diễn bằng khoảng cách Euclidean như sau: 
    d = sqrt((x2 - x1)² + (y2 - y1)²)
  • Với vector 3 chiều: 
    d = sqrt((x2 - x1)² + (y2 - y1)² + (z2 - z1)²)
  • Tương tự cho các chiều cao hơn.
  • Trong thực tế khi số chiều vượt quá 200, giá trị khoảng cách giữa các vector được tính bằng phương pháp Euclidean có thể trở nên gần như bằng nhau (hiện tượng Lời nguyền chiều không gian - Curse of Dimensionality), làm giảm hiệu quả tìm kiếm.
  • Để giải quyết vấn đề này, các phương pháp khác như Cosine similarity thường được sử dụng để đánh giá độ tương đồng giữa các vector trong không gian nhiều chiều.
  • Thay vì đo khoảng cách, cosine similarity đo góc giữa hai vector. Góc càng nhỏ thì độ tương đồng càng cao. Góc 180 độ tương ứng với ngữ nghĩa hoàn toàn khác nhau - điều mà khoảng cách Euclidean không thể phản ánh.

Vector Database Là Gì?

Vector database là database được tối ưu để lưu và truy vấn dữ liệu dạng:

[0.1, 0.5, 0.9, 0.2, 0.8,...]

Mỗi đoạn text, ảnh hoặc audio có thể được mã hóa thành một dãy số để đo mức độ giống nhau.

Khác với database truyền thống lưu dữ liệu dạng số hay chữ,... Vector database lưu vector và tài liệu gốc của vector đó.

Thao Tác Với ChromaDB

Tạo, quản lý database, collection và record

ChromaDB: Database

Có 2 cách tạo database: PersistentClient và Client

  • PersistentClient: lưu dữ liệu vào ổ cứng, có thể mở lại và thấy dữ liệu cũ.
  • Client: chỉ lưu trong RAM, khi tắt chương trình sẽ mất hết dữ liệu.
import chromadb

# Tạo PersistentClient để lưu dữ liệu vào ổ cứng
client = chromadb.PersistentClient(
    path="./chroma_data"
)

# Tạo Client để lưu dữ liệu chỉ trong RAM
client = chromadb.Client()

Giải thích

  • PersistentClient(...): mở hoặc tạo database local.
  • path: thư mục ChromaDB dùng để lưu file dữ liệu.
  • Nếu chạy lại chương trình với cùng path, bạn sẽ thấy lại dữ liệu cũ.

ChromaDB: Collection

  • Collection tương đương một bảng (table) trong các hệ quản trị cơ sở dữ liệu quan hệ đã biết (MySQL, MSSQL, PostgreSQL, v.v.)
  • Mỗi collection lưu một nhóm tài liệu riêng (FAQ, bài giảng, bài báo,...)
collection = client.get_or_create_collection(
    name="khoa_hoc_ai"
)

Giải thích

  • client: database đã kết nối ở bước trước.
  • get_or_create_collection: nếu collection đã tồn tại thì lấy ra, nếu chưa có thì tạo mới.
  • name: tên collection (tương đương tên bảng).

ChromaDB: Record

  • Record là đơn vị dữ liệu cơ bản trong ChromaDB, tương đương một hàng (row) trong bảng của các hệ quản trị cơ sở dữ liệu quan hệ.
  • Mỗi record là một cấu trúc dữ liệu cố định bao gồm 4 thuộc tính (cột): id, embedding, document và metadata.
  • Trong đó:
    • id: khóa chính.
    • embedding: vector đại diện cho ý nghĩa của văn bản.
    • document: nội dung gốc của vector.
    • metadata: dữ liệu phụ như topic, source, chapter, level,... để lọc, gắn nhãn và mô tả thêm.
  • Cấu trúc của record là cố định, không thể thay đổi.
# Thêm record
collection.add(
    ids=["doc_1"],
    embeddings=[vector],
    documents=["ChromaDB là database cho vector"],
    metadata=[{"source": "website"}]
)

# Sửa record, ids là bắt buộc
collection.update(
    ids=["doc_1"],
    embeddings=[new_vector],
    documents=["ChromaDB có collection và record"],
)

# Xóa record
collection.delete(ids=["doc_1"])

ChromaDB: Record

Một số lưu ý

  • Các params khi thêm/sửa/xóa record như ids, embeddings, documents, metadatas đều là danh sách (list).
  • Các phần tử cùng vị trí trong các danh sách sẽ ghép thành một record hoàn chỉnh.
  • Thuộc tính document của record không bắt buộc vì vector có thể phản ánh ngữ nghĩa của hình ảnh hoặc audio.
  • Thuộc tính metadata của mỗi record là nơi lưu thông tin bổ sung cho record đó, được dùng để lọc, tổ chức và quản lý dữ liệu tốt hơn.
  • Thuộc tính metadata khi update sẽ bị ghi đè hoàn toàn.
# Ví dụ về metadata
collection.add(
    ids=["doc_1"],
    embeddings=[vector],  # Vector tạo bởi OpenAI API
    documents=["ChromaDB là database cho vector"],
    metadatas=[{
        "topic": "AI",
        "title": "Giới thiệu ChromaDB",
        "user_id": "user_001",
        "created_at": "2026-03-12T10:00:00Z"
    }]
)

ChromaDB: Query Record

# query_vector được tạo bởi OpenAI API trước
results = collection.query(
    query_embeddings=[query_vector],
    where={"user_id": "user_001"},
    n_results=3
)

print(results)

Giải thích

  • collection.query(...): tìm các record gần nhất với vector câu hỏi.
  • query_embeddings: vector của câu hỏi cần truy vấn.
  • where: điều kiện lọc theo metadata. Ở đây chỉ tìm các record có metadata user_id bằng user_001. Điều kiện where được ưu tiên tìm kiếm trước.
  • n_results=3: lấy ra top 3 kết quả gần nhất.
  • Thứ tự kết quả được sắp xếp theo khoảng cách ngữ nghĩa, với kết quả có nghĩa gần nhất được liệt kê đầu tiên.

RAG Là Gì?

RAG (Retrieval-Augmented Generation) là cách kết hợp:

  • Retrieval: tìm đoạn tài liệu liên quan nhất trong kho tài liệu (knowledge base).
  • Generation: đưa các đoạn đó cho LLM để sinh câu trả lời có căn cứ.

Lợi ích chính:

  • Giảm ảo giác vì câu trả lời dựa trên tài liệu thật.
  • Cho phép cập nhật tri thức mà không cần huấn luyện lại model.
  • Dễ kiểm tra nguồn thông tin bằng metadata và document trả về.

Chunking, Overlap Và Cách Chọn Tham Số

Chunking là chia tài liệu thành đoạn nhỏ để embedding rõ nghĩa hơn.

Overlap là phần chồng lấn giữa hai chunk liên tiếp để không mất ngữ cảnh.

Ví dụ:

  • Nếu không có overlap, câu "Rắn là loài bò sát không chân" có thể bị cắt thành "Rắn là loài bò" và "sát không chân", làm mất hoặc sai ý nghĩa của câu ban đầu.
  • Với chunking và overlap, ta có thể tạo ra các chunk như "Rắn là loài bò sát" và "bò sát không chân", giữ được ngữ nghĩa đầy đủ hơn.

Thông thường, phần overlap có kích thước từ 10% đến 20% kích thước chunk. Kích thước overlap quá nhỏ dễ mất ngữ cảnh, kích thước quá lớn gây tốn tài nguyên. Dưới đây là một số mẫu kích thước tham khảo:

  • FAQ ngắn: chunk 250-400 token, overlap 30-60. Hoặc chunk theo cả câu.
  • Bài viết blog: chunk 400-700 token, overlap 60-120.
  • Tài liệu kỹ thuật dài: chunk 700-1200 token, overlap 100-180.
  • PDF scan nhiều nhiễu: chunk nhỏ hơn để giảm lỗi ngữ nghĩa.

Code Mẫu: Triển Khai Hàm Chunk Theo Ký Tự

def chunk_by_chars(text, chunk_size=1200, overlap=150):
    chunks = []
    start = 0

    while start < len(text):
        end = start + chunk_size
        piece = text[start:end].strip()
        if piece:
            chunks.append(piece)

        if end >= len(text):
            break
        start = end - overlap

    return chunks

Khi nào dùng?

  • Triển khai nhanh và dễ.
  • Không muốn cài thêm package bên ngoài.
  • Nhược điểm: không kiểm soát chính xác số token cho model.
  • Phù hợp bản demo nhanh và dữ liệu nhỏ.

Code Mẫu: Chunk Bằng tiktoken

import tiktoken

def chunk_by_tokens(text, chunk_size=500, overlap=80,
                    model="gpt-5-mini"):
    enc = tiktoken.encoding_for_model(model)
    token_ids = enc.encode(text)
    chunks = []
    step = chunk_size - overlap

    for i in range(0, len(token_ids), step):
        piece_ids = token_ids[i:i + chunk_size]
        if not piece_ids:
            continue
        chunks.append(enc.decode(piece_ids).strip())

    return [c for c in chunks if c]

Ưu điểm

  • Chunk theo token gần với cách model xử lý thật.
  • Ổn định hơn khi embedding, xử lý truy vấn tốt hơn .
  • Nên dùng khi làm tool chạy lâu dài hoặc dữ liệu lớn.

def embed_chunks(chunks, model="text-embedding-3-small"):
    response = openai_client.embeddings.create(
        model=model,
        input=chunks
    )
    return [item.embedding for item in response.data]

def make_ids(n):
    return [str(uuid.uuid4()) for _ in range(n)]

Source Quy Trình (5/7): Nạp Dữ Liệu Vào ChromaDB

def index_file(path, topic="general"):
    chunks = build_chunks(path)
    vectors = embed_chunks(chunks)
    ids = make_ids(len(chunks))

    metadatas = [
        {
            "topic": topic,
            "source_file": Path(path).name,
            "chunk_idx": i
        }
        for i in range(len(chunks))
    ]

    collection.add(
        ids=ids,
        embeddings=vectors,
        documents=chunks,
        metadatas=metadatas
    )

def search(question, top_k=3, topic=None):
    q_vec = openai_client.embeddings.create(
        model="text-embedding-3-small",
        input=question
    ).data[0].embedding

    kwargs = {
        "query_embeddings": [q_vec],
        "n_results": top_k,
        "include": ["documents", "metadatas", "distances"]
    }
    if topic:
        kwargs["where"] = {"topic": topic}

    result = collection.query(**kwargs)
    return result["documents"][0], result["metadatas"][0]

        if question.lower() == "exit":
            break

        docs, metas = search(question, top_k=3, topic=topic)
        print("\nTop kết quả:")
        for i, (doc, meta) in enumerate(zip(docs, metas), start=1):
            print(f"[{i}] {meta['source_file']} - chunk {meta['chunk_idx']}")
            print(doc[:280], "...\n")

if __name__ == "__main__":
    main()
